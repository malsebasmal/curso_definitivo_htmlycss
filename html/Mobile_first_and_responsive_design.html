Mobile first and Responsive design:

Ambas formas/conceptos sirven para crear código que se pueda adaptar a los distintos tipos de resoluciones que existen en los distintos tipos de dispositivos. Es crear páginas webs que se adapten completamente a cualquier dispositivo sin importar su tamaño.
Comprendido esto, tenemos que tener en cuenta 2 conceptos:

-Breakpoints:
Estos "breakpoints" no son más que tamaños específicos que indicamos a los "Media Queries" donde dentro de estos pondremos cierto código que se ejecutara una vez el tamaño indicado en el breakpoint sea el que muestre en la pantalla del usuario. Por ejemplo:
@media (max-width: 768px) { Código que se aplicara cuando la pantalla tenga la resolución indicada }
En este caso le estamos indicando al navegador que cuando la página tenga el tamaño máximo de 786px aplicara los cambios.

-Media Queries:
Las media queries son una característica de CSS que permite aplicar estilos diferentes según ciertas características del dispositivo o de la pantalla. Las media queries se utilizan junto con los breakpoints para adaptar el diseño en diferentes situaciones. Por ejemplo:
@media (max-width: 768px) { Estilos aplicados cuando el ancho de la pantalla es menor o igual a 768px }


Entendido esos 2 conceptos, tenemos que entender la diferencia entre Mobile First y Responsive Design:

-Mobile First:
Como su nombre lo indica, esta manera de escribir código adaptable a distintos tamaños de resoluciones para los distintos dispositivos funciona EMPEZANDO por DESARROLLAR PRIMERO (first) la página web para un CELULAR (Mobile). Luego se va escalando sucesivamente en las distintas resoluciones de dispositivos más grandes hasta llegar a Desktop (pc) la cual sería la más grande.
Es decir de pequeño a más grande.

-Responsive Design:
En Responsive Design se empieza al contrario. Desde lo más grande hasta ir adaptando el código a los dispositivos más pequeños.
